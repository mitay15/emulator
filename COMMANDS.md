
# 🧱 **Команды разработки (Makefile) — Полное описание**

## 🔍 **Тестирование**

### **`make test`**
Запускает все тесты в тихом режиме (`pytest -q`).  
Используется для быстрого локального прогона.

---

### **`make testv`**
Запускает тесты подробно (`pytest -vv`).  
Показывает каждый тест, стек‑трейсы, метки.

---

### **`make smoke`**
Запускает только smoke‑тесты (`pytest -m smoke`).  
Проверяет:

- Auto‑GA v3 запускается  
- compare_runner работает  
- pipeline не падает  

Используется перед релизом.

---

### **`make integration`**
Запускает интеграционные тесты (`pytest -m integration`).  
Проверяет:

- pipeline на реальных блоках  
- работу с cache  
- корректность вычислений  

---

### **`make visual`**
Запускает визуальные тесты (`pytest -m visual`).  
Генерирует:

- heatmap  
- predBG графики  

Используется для визуальной диагностики.

---

### **`make unit`**
Запускает юнит‑тесты (`pytest -m unit`).  
Проверяет:

- kotlin_parser  
- мелкие функции  
- структуры данных  

---

# ⚙️ **Основные команды проекта**

### **`make ga`**
Запускает Auto‑GA v3 через `run.py ga`.  
Используется для оптимизации профиля.

---

### **`make gui`**
Запускает Streamlit GUI (`run.py gui`).  
Открывает визуальный интерфейс эмулятора.

---

### **`make compare`**
Запускает compare_runner (`run.py compare`).  
Сравнивает Python ↔ AAPS логи, выводит статистику.

---

### **`make inputs`**
Генерирует `inputs_before_algo_block_*.json` из логов.  
Используется для подготовки данных для тестов и анализа.

---

# 🧹 **Форматирование и линтинг**

### **`make format`**
Автоматически исправляет стиль кода:

- `ruff check --fix` — исправляет ошибки стиля  
- `black .` — форматирует код  
- `autoflake` — удаляет неиспользуемые импорты  

Используется перед коммитом.

---

### **`make lint`**
Проверяет стиль кода, но **не исправляет**:

- `ruff check .`  
- `black --check .`  
- `autoflake -r .`  

Используется в CI или перед PR.

---

# 📘 **Документация**

### **`make docs`**
Генерирует HTML‑документацию API через pdoc:

```
docs/
    index.html
    aaps_emulator/
```

Используется для:

- документации проекта  
- публикации на GitHub Pages  
- внутреннего анализа API  

---

# 🧹 **Очистка**

### **`make clean`**
Удаляет временные файлы:

- `__pycache__`  
- `.pytest_cache`  
- `dist/`, `build/`  
- `data/cache/*`  
- `data/reports/*`  

Используется перед релизом или CI.

---

# 🚀 **Релизы**

### **`make release VERSION=x.y.z`**
Автоматически:

1. Создаёт git‑tag `vX.Y.Z`  
2. Пушит его на GitHub  
3. GitHub Actions запускает:
   - сборку wheel  
   - создание GitHub Release  
   - загрузку артефактов  

Используется для публикации новых версий.

Пример:

```
make release VERSION=0.2.0
```

# 🧱 **Команды run.py — Полное описание**

`run.py` — это **универсальный CLI‑вход** для проекта.  
Он позволяет запускать ключевые функции без знания внутренних модулей.

Запуск:

```
python run.py <команда>
```

---

# 🚀 **1. `python run.py ga` — запуск Auto‑GA v3**

### Что делает:
- запускает генетический оптимизатор Auto‑GA v3  
- проходит несколько поколений  
- выводит лучший fitness  
- используется для подбора профиля под реальные данные  

### Аргументы:
```
--generations N
--population N
--elitism N
--auto
```

Пример:

```
python run.py ga --generations 40 --population 80 --auto
```

---

# 🔍 **2. `python run.py compare` — запуск compare_runner**

### Что делает:
- прогоняет сравнение Python ↔ AAPS  
- читает логи из `data/logs/`  
- генерирует статистику расхождений  
- выводит количество блоков  

### Аргументы:
```
--fast
--extract-clean
```

Пример:

```
python run.py compare --fast
```

---

# 📦 **3. `python run.py inputs` — генерация inputs_before_algo_block_*.json**

### Что делает:
- читает логи AAPS  
- создаёт файлы:

```
data/cache/inputs_before_algo_block_*.json
```

### Аргументы:
```
--logs DIR
--out DIR
```

Пример:

```
python run.py inputs --logs mylogs/
```

---

# 🖥 **4. `python run.py gui` — запуск Streamlit GUI**

### Что делает:
- запускает Streamlit приложение  
- открывает визуальный интерфейс:
  - predBG графики  
  - AutoISF pipeline  
  - Auto‑GA v3  
  - отчёты  

### Особенности:
- автоматически проверяет, установлен ли streamlit  
- не падает, если streamlit отсутствует  

---

# 🧪 **5. `python run.py test` — запуск pytest**

### Что делает:
- запускает все тесты (`pytest -q`)  
- используется как удобная обёртка  

---

# 🔧 **6. `python run.py prepare` — полная подготовка проекта**

### Что делает:
1. Генерирует inputs  
2. Запускает compare_runner  
3. Запускает тесты  

### Используется:
- перед релизом  
- после больших изменений  
- для CI  

Пример:

```
python run.py prepare
```
